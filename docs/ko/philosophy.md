# 철학

이 문서는 Codingbuddy의 비전, 핵심 신념, 설계 원칙을 설명합니다.

## 비전

**모든 AI 어시스턴트를 위한 단일 AI 코딩 규칙 소스**

오늘날 개발 팀은 Cursor, Claude Code, GitHub Copilot 등 여러 AI 코딩 도구를 사용합니다. 각 도구마다 고유한 설정 형식이 있어 다음과 같은 문제가 발생합니다:

- 여러 설정 파일에 규칙 중복
- 사용하는 AI 도구에 따라 일관성 없는 코딩 표준
- 규칙 업데이트 시 유지보수 부담

Codingbuddy는 모든 AI 어시스턴트와 호환되는 통합 규칙 시스템을 제공하여 이 문제를 해결합니다.

## 핵심 신념

### 1. AI 도구에 구애받지 않는 규칙

규칙은 한 번 작성하면 어디서나 작동해야 합니다. 벤더 종속 없이, 핵심 규칙에 도구별 문법 없이. 각 AI 도구는 경량 어댑터를 통해 공통 형식에 맞춰집니다.

### 2. 점진적 공개

간단하게 시작하고, 필요할 때 깊이 들어갑니다:

- **빠른 시작**: `npx codingbuddy init`으로 2분 만에 실행
- **설정**: 기술 스택, 아키텍처, 컨벤션 커스터마이징
- **전문가 에이전트**: 도메인 전문가 접근 (보안, 성능, 접근성)
- **완전한 커스터마이징**: 프로젝트별 규칙으로 확장

### 3. 설정보다 규약

대부분의 프로젝트에서 작동하는 합리적인 기본값:

- PLAN → ACT → EVAL 워크플로우
- TDD 우선 개발 방식
- 80% 이상 테스트 커버리지 목표
- SOLID 원칙과 클린 코드

변경이 필요한 부분만 오버라이드합니다.

### 4. 커뮤니티 주도 표준

최고의 관행은 실제 경험에서 나옵니다:

- 규칙은 프로덕션 코드베이스의 검증된 패턴 기반
- 전문가 에이전트는 실무자의 도메인 전문성을 인코딩
- 오픈 소스이며 기여에 열려 있음

## 설계 원칙

### 단일 소스

```
packages/rules/.ai-rules/           ← 권위 있는 소스
├── rules/           ← 핵심 규칙 (워크플로우, 품질, 프로젝트)
├── agents/          ← 전문가 지식
└── adapters/        ← 도구별 통합 가이드
```

모든 AI 도구 설정이 `packages/rules/.ai-rules/`를 참조합니다. 한 번 수정하면 모든 도구에 적용됩니다.

### 관심사의 분리

| 레이어 | 목적 | 형식 |
|--------|------|------|
| **Rules** | 무엇을 할지 (워크플로우, 품질 표준) | Markdown |
| **Agents** | 누가 무엇을 아는지 (전문가 지식) | JSON |
| **Adapters** | 어떻게 통합할지 (도구별 설정) | Markdown |

이러한 분리를 통해:

- 규칙이 도구 지원과 독립적으로 발전 가능
- 핵심 규칙 변경 없이 새 에이전트 추가 가능
- 기존 규칙 수정 없이 새 도구 지원 가능

### 복잡성보다 확장성

시스템은 설정이 아닌 확장을 위해 설계되었습니다:

- JSON 파일 생성으로 새 전문가 에이전트 추가
- 어댑터 가이드 작성으로 새 도구 지원
- 핵심 규칙 수정 없이 프로젝트별 컨텍스트 포함

간단한 것은 간단하게. 복잡한 것은 가능하게.

## 워크플로우 모델

Codingbuddy는 AI 기반 개발을 위한 구조화된 워크플로우를 도입합니다:

```
PLAN → ACT → EVAL
```

### PLAN 모드 (기본)

- 요구사항 이해
- 구현 접근 방식 설계
- 위험과 엣지 케이스 식별
- 코드 변경 없음

### ACT 모드

- 계획 실행
- TDD 따르기: Red → Green → Refactor
- 점진적이고 테스트된 변경

### EVAL 모드

- 구현 품질 검토
- 개선점 식별
- 리팩토링 기회 제안

이 워크플로우는 AI 어시스턴트가 적절한 계획 없이 바로 코드로 뛰어드는 일반적인 함정을 방지합니다.

## Codingbuddy가 아닌 것

- **코드 생성기가 아님**: 생성된 코드가 아닌 규칙과 컨텍스트를 제공
- **인간 판단의 대체제가 아님**: 개발자 의사결정을 대체하지 않고 보강
- **만능 솔루션이 아님**: 프로젝트별로 커스터마이징되도록 설계됨
