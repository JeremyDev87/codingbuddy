# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of Codingbuddy seriously. If you discover a security vulnerability, please follow these steps:

### Do

- Report vulnerabilities privately via [GitHub Security Advisory](https://github.com/JeremyDev87/codingbuddy/security/advisories/new)
- Provide detailed information about the vulnerability
- Allow reasonable time for us to address the issue before public disclosure

### Don't

- Open public GitHub issues for security vulnerabilities
- Exploit vulnerabilities beyond what is necessary to demonstrate the issue
- Share vulnerability details with others before it has been addressed

### What to Include

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Initial Response:** Within 48 hours
- **Status Update:** Within 7 days
- **Resolution Target:** Within 30 days for critical issues

### Recognition

We appreciate security researchers who help keep Codingbuddy safe. Contributors who report valid security issues will be acknowledged in our release notes (unless they prefer to remain anonymous).

## Security Best Practices

When using Codingbuddy:

- Keep dependencies up to date
- Review AI-generated rules before applying them
- Use environment variables for sensitive configuration
- Follow the principle of least privilege when configuring MCP servers
